# Cortex — Office Hours Design Doc

**Дата:** 03.06.2026  
**Автор:** Евгений + Hermes (gstack-office-hours, Startup Mode)

---

## 1. Status Quo — Что болит

| Проблема | Как сейчас |
|---|---|
| Задачи разбросаны | Redmine + Apple Reminders + Telegram-чаты + память |
| Обучение без структуры | Начал → посмотрел чуть-чуть → бросил |
| Нет roadmap | 120+ курсов, непонятно в каком порядке и зачем |
| Инфошум | Много воды, нет выжимок под задачу |
| Расшифровки — мёртвый груз | Hermes делает, но всё лежит в Obsidian неиспользуемым |
| Практика не привязана | Нет связи «видео → задание → мой код → фидбек» |
| Мелкий контент теряется | Статьи, подкасты, письма — полезные инсайты не сохраняются |

## 2. Desperate Specificity — Кому это нужно

**Прямо сейчас — мне.** Open-source проект «для себя», который может пригодиться другим.

## 3. Narrowest Wedge

**Фаза 1 (2 недели): Курс + Hermes-тьютор**
- Выжимка из видео: ключевые моменты с таймкодами, без воды
- Практическая часть: квиз + ссылка на учебный GitHub-проект
- Hermes ревьюит PR в учебный проект
- Диалог с Hermes по каждому вопросу с контекстными ссылками

**Фаза 2: Единый inbox задач** (Redmine + Reminders + Telegram)

## 4. Observations — Реальное поведение

- Курсы начинаются и бросаются. Причина: нет практики и нет видимого прогресса.
- Obsidian хорош как хранилище, но плох как интерфейс обучения.
- Hermes может расшифровывать и делать выжимки, но без структуры это бесполезно.
- Мелкие статьи/подкасты/письма содержат инсайты ценнее целых курсов, но не сохраняются.

## 5. Идеальный опыт (час с Cortex)

1. Открываю Cortex → вижу курс, на котором остановился
2. **Выжимка видео:** ключевые концепции с объяснением, таймкоды к частям без воды
3. **Практика:** квиз по материалу + задание в учебном GitHub-репо
4. **Фидбек от Hermes:** разбор прошлых занятий, ревью моих PR
5. **Диалог:** по каждому вопросу — обсуждение с Hermes, ссылки на документацию/статьи
6. Прогресс виден → хочется вернуться завтра

## 6. Future-fit — Что должно быть заменяемым

| Компонент | Должен быть pluggable |
|---|---|
| AI-агент | Да — не только Hermes (OpenClaw, Claude Code, etc.) |
| Источники курсов | Да — локальные файлы, YouTube, любые видео |
| Inbox | Да — статьи, подписки из почты, подкасты, мелкие видео |
| Обработка inbox | Авто-тегирование → раскладывание по темам → извлечение знаний → встраивание в учебный план |
| Форматы контента | Статьи, документация, подкасты (выжимки), видео |
| Языки | Русский + английский |
| База знаний | Markdown-first (Obsidian-совместимо), SQLite для структурированных данных |

## Архитектурные принципы

- **Self-hosted + local-first** — как приложения для Hermes/OpenClaw, но human-oriented
- **Hermes — тьютор, не просто расшифровщик** — proactive: выжимки, ревью кода, ссылки, диалог
- **Inbox → Knowledge Pipeline**
- **Progress visibility** — видимый прогресс мотивирует возвращаться
- **Open-source** — архитектура позволяет форкнуть и заменить любой компонент

## Стек (из памяти)

- TypeScript (full-stack, один язык)
- Next.js или Remix
- SQLite (структурированные данные)
- Markdown (Obsidian vault для базы знаний)
- Hermes интеграция через terminal/file tools
